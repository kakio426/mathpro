# Teacher Field Test Playbook

이 문서는 실제 선생님에게 수학프로 v0를 보여주고 피드백을 받을 때 쓰는 운영 대본이다. 목표는 기능 설명이 아니라 `교사가 자료를 만들고, 학생에게 배포하고, 과정 리포트를 읽는 흐름`이 자연스러운지 확인하는 것이다.

## 테스트 목표

- 선생님이 수학프로를 `문제풀이 사이트`가 아니라 `움직이는 수업자료 제작/배포 도구`로 이해하는지 확인한다.
- Gemini/LLM에서 만든 HTML을 붙여넣는 방식이 부담스럽지 않은지 확인한다.
- 참여 코드, QR, 링크 공유가 교실 상황에서 충분히 빠른지 확인한다.
- 학생 활동 후 교사 리포트의 문장이 다음 수업 준비에 바로 도움이 되는지 확인한다.

## 권장 진행 시간

- 전체 25-35분
- 제품 소개 3분
- 자료 제작 8분
- 학생 참여 시뮬레이션 7분
- 리포트 확인 7분
- 인터뷰 5-10분

## 준비물

- 배포 URL 또는 로컬 production URL
- Supabase가 연결된 테스트 환경
- 테스트용 Gemini/LLM 접속 환경
- 교사용 노트북 1대
- 학생 화면 확인용 모바일 또는 태블릿 1대 이상
- 테스트 후 정리할 QA assignment code 목록

## 진행 대본

1. 제작실 열기
- 선생님에게 `/` 제작실을 보여준다.
- "여기는 문제를 푸는 곳이 아니라, 선생님이 움직이는 수업자료를 만드는 작업대입니다."라고 설명한다.
- 선생님이 `바로 쓸 수 있는 예시` 중 하나를 직접 선택하게 한다.

2. AI 요청문 확인
- `Gemini에 보낼 요청문`을 복사한다.
- Gemini/LLM에 붙여넣고 HTML 결과를 받는다.
- 시간이 부족하면 기본 예시 HTML을 그대로 사용한다.

3. 자료 원본 붙여넣기
- 생성된 HTML을 `자료 원본 붙여넣기` 영역에 넣는다.
- `자료 문서 만들기`를 누르고 안전 검사 결과를 확인한다.
- 경고가 나오면 선생님이 이해할 수 있는 문구인지 묻는다.

4. 발행과 공유
- `발행하기`를 누른다.
- 참여 코드, QR, 학생 링크가 보이는지 확인한다.
- 학생 기기에서 QR 또는 `/join` 코드 입력으로 들어간다.

5. 학생 활동
- 학생 화면에서 조작, 제출, 힌트, 다시 시도, 완료를 한 번씩 만든다.
- 학생 화면에 개발자용 설명이 보이지 않는지 확인한다.
- 완료 후 학생 리포트로 이동하는지 확인한다.

6. 교사 리포트 확인
- `/teacher/assignments/[assignmentId]`에서 교사 리포트를 연다.
- 4블록 리포트, 학생별 참여, 활동별 로그를 확인한다.
- 선생님에게 "이 문장을 보고 다음 수업에서 무엇을 할지 떠오르나요?"라고 묻는다.

## 인터뷰 질문

- 처음 화면을 보고 무엇을 하는 서비스인지 바로 이해했나요?
- HTML, AI 요청문, 안전 검사, 조작 기록이라는 말이 부담스럽거나 낯설었나요?
- 자료 제작 과정에서 가장 막히는 지점은 어디였나요?
- 학생에게 공유하는 방식은 수업 중에 쓰기 충분히 빠른가요?
- 리포트 문장이 실제 수업 보완에 도움이 되나요?
- 리포트에서 더 보고 싶은 것은 정답률, 학생별 이름표, 활동별 시간, 오개념 설명 중 무엇인가요?
- 이 서비스를 다시 쓰려면 꼭 필요한 기능은 무엇인가요?

## 관찰 체크리스트

- 선생님이 예시 템플릿을 스스로 고르는가
- AI 요청문을 복사해 Gemini/LLM에 붙여넣는 흐름을 이해하는가
- HTML 결과물을 어디에 붙여넣어야 하는지 헷갈리지 않는가
- 안전 검사 경고를 보고 다음 행동을 알 수 있는가
- 참여 코드/QR/링크 중 어떤 공유 방식을 선호하는가
- 학생 화면에서 활동 자체가 충분히 크게 보이는가
- 리포트 문장이 내부 용어 없이 읽히는가
- 다음 수업 액션이 실제 수업 질문으로 바뀌는가

## 실패 대응표

| 상황 | 먼저 확인할 것 | 대응 |
| --- | --- | --- |
| 자료 문서 만들기가 실패함 | HTML이 비어 있거나 너무 짧지 않은지 확인 | 예시 템플릿으로 되돌린 뒤 다시 생성 |
| 안전 검사에서 발행 차단 | 외부 이미지, 외부 스크립트, form, fetch, localStorage 포함 여부 확인 | Gemini에 "외부 리소스 없이 단일 HTML로 다시 만들어줘"라고 요청 |
| 학생 화면이 비어 있음 | HTML에 `<body>` 내용과 script 오류가 있는지 확인 | 예시 템플릿으로 교체해 학생 흐름부터 시연 |
| 조작 기록이 쌓이지 않음 | HTML 안에 `window.parent.postMessage`와 `source: "mathpro-html-activity"`가 있는지 확인 | 제작실 도움말의 이벤트 예시를 Gemini에 추가로 붙여넣기 |
| 완료 후 이동하지 않음 | `complete` 이벤트가 있는지 확인 | 활동 완료 버튼에 `send("complete", {...})`가 들어가도록 수정 |
| 교사 리포트가 비어 있음 | 학생 세션이 완료되었는지, 이벤트가 저장되었는지 확인 | 학생 화면에서 제출/완료를 다시 수행 |
| Supabase 저장 실패 | env 3종과 migrations 적용 여부 확인 | `docs/product/h7-release-checklist.md`의 Supabase Checklist 재확인 |

## 테스트 후 정리

- 생성된 참여 코드와 assignment id를 기록한다.
- QA 데이터 삭제가 필요하면 Supabase에서 해당 assignment/session/event/report row를 직접 정리한다.
- 피드백은 `화면 이해`, `제작 막힘`, `공유 막힘`, `리포트 유용성`, `필수 다음 기능`으로 분류한다.
