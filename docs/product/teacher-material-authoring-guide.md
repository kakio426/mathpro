# Teacher Material Authoring Guide

수학프로의 자료 제작 기준은 `AI가 초안을 만들고, 선생님이 수업 의도에 맞게 검토한 뒤 발행한다`이다. 이 문서는 선생님 테스트 전에 운영자가 자료 제작을 도울 때 쓰는 가이드다.

## 좋은 자료의 기준

- 학생이 최소 한 번 이상 직접 눌러 보거나 움직여 볼 수 있다.
- 정답 확인 전에 관찰, 예상, 설명을 유도한다.
- 한 화면 안에서 활동 목표가 분명하다.
- 외부 이미지, 외부 스크립트, 로그인이 필요 없다.
- 활동 완료 버튼이 있고, 완료 후 수학프로 리포트로 넘어간다.

## 추천 제작 순서

1. 제작실에서 예시 템플릿을 고른다.
2. 개념과 수업 목표를 실제 차시에 맞게 바꾼다.
3. AI 요청문을 복사해 Gemini/LLM에 붙여넣는다.
4. 생성된 HTML 전체를 수학프로에 붙여넣는다.
5. 미리보기에서 학생 화면을 확인한다.
6. 안전 검사 결과를 보고 차단 항목이 없으면 발행한다.
7. 학생 링크 또는 QR로 1명에게 먼저 테스트한다.

## Gemini/LLM에 추가하면 좋은 요청

```text
초등학생이 모바일/태블릿에서 터치로 조작할 수 있게 만들어줘.
문제풀이보다 직접 만져 보고 관찰하는 활동으로 만들어줘.
외부 이미지, 외부 스크립트, CDN, fetch, localStorage, form은 사용하지 마.
학생의 선택, 힌트, 재시도, 제출, 완료를 window.parent.postMessage로 보내줘.
완료 버튼을 만들고 complete 이벤트를 반드시 보내줘.
오답 제출에는 misconceptionSignal을 학생 사고를 설명하는 짧은 영어 키로 넣어줘.
```

## 자료 유형별 팁

### 분수 막대

- 전체가 무엇인지 화면에 먼저 보여준다.
- 조각 수와 선택한 부분 수를 학생이 직접 바꾸게 한다.
- 오답 피드백은 "몇 조각으로 나누었나요?"처럼 전체와 등분할을 다시 보게 한다.

### 수직선

- 0과 1 사이가 같은 간격으로 나뉘었는지 강조한다.
- range slider나 드래그 마커를 사용하면 모바일에서 안정적이다.
- 오답 피드백은 "0에서 몇 칸 이동했나요?"처럼 간격 세기를 유도한다.

### 분류/매칭

- 선택지를 너무 많이 만들지 않는다. MVP 테스트에서는 2-4개가 적당하다.
- 학생이 고른 이유를 말할 수 있도록 제출 후 한 문장 질문을 넣는다.
- 같은 전체/다른 전체, 분모/분자처럼 헷갈리기 쉬운 개념을 짝짓기에 적합하게 쓴다.

## 피해야 할 자료

- 단순 4지선다만 있는 자료
- 긴 설명문만 있고 학생 조작이 없는 자료
- 외부 사이트로 이동하거나 이미지를 불러오는 자료
- 학생 이름, 전화번호, 사진, 위치 같은 개인정보를 요구하는 자료
- 완료 버튼이 없어서 리포트로 넘어갈 수 없는 자료

## 발행 전 셀프 체크

- 미리보기에서 버튼이나 조작 요소가 잘 보인다.
- 학생이 무엇을 해야 하는지 첫 문장으로 알 수 있다.
- 제출 버튼과 활동 완료 버튼이 있다.
- 안전 검사 상태가 `통과` 또는 이해 가능한 `확인 필요`이다.
- 학생에게 공유할 참여 코드/QR/링크가 생성된다.
