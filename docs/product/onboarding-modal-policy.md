# Onboarding Modal Policy

수학프로 v0의 안내 모달은 사용자를 붙잡아 두는 튜토리얼이 아니라, 첫 화면에서 무엇을 하면 되는지 빠르게 알려주는 짧은 길잡이다.

## 적용 화면

| 화면 | 자동 안내 | 이유 |
| --- | --- | --- |
| 제작실 `/` | 첫 방문 자동 표시 | 선생님이 HTML 붙여넣기 흐름을 처음 이해해야 함 |
| 학생 참여 `/join` | 첫 방문 자동 표시 | 학생이 로그인 없이 코드로 들어가는 방식을 바로 알아야 함 |
| 학생 실행 `/play/[code]` | 첫 방문 자동 표시 | 활동 무대와 저장 흐름을 짧게 안내해야 함 |
| 교사 리포트 `/teacher/assignments/[id]` | 첫 방문 자동 표시 | 정답률보다 과정 신호를 읽는 화면임을 알려야 함 |

## 저장 정책

- 화면별 `storageKey`를 사용한다.
- 사용자가 안내를 닫으면 `seen`을 저장해 다음 방문부터 자동으로 열지 않는다.
- 사용자가 `이 화면 안내 다시 자동으로 보지 않기`를 선택하고 닫으면 `dismissed`를 저장한다.
- 자동 안내가 꺼져도 화면의 안내 버튼은 남겨 둔다.

## 문구 원칙

- 학생 화면에서는 제작 도구, HTML, postMessage 같은 개발자 용어를 노출하지 않는다.
- 교사 화면에서는 낯선 용어를 숨기지 않고 쉬운 설명을 함께 제공한다.
- 한 단계는 한 가지 행동만 설명한다.
- 안내는 다음 행동으로 이어져야 한다.
